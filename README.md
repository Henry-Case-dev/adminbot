# AdminBot — ваш персональный цифровой абьюзер

*Элегантное решение для автоматизации токсичного общения в чатах.*

## 🚀 Возможности

### Костик (F7)
На каждое сообщение — случайное оскорбление из пула. Вероятность настраивается: от «всегда» до «никогда».

### Слава (F3/F4/F5)
Трёхуровневая система:
- Просто написал → **«пошёл нахуй»** (catch-all)
- Упомянул «куча»/«кучи»/«кучу» → **«ДАЛБАЕБ»** (F4)
- Ляпнул про дроны/войну → **«трясло ебаное»** (F5)
- Каждое 5-е сообщение → **GIF** со славным членом (F3)

### Слава: Dead Page (F2 v2)
- Кто-то делает репост из канала **@d_pages** → бот выбирает случайный пост из своего приватного канала и пересылает его в чат
- Если канал недоступен → fallback: случайная картинка + текст из папки `media/dead_page/`
- При входе Славы в чат → «ДОЛБОЕВ ВЕРНУЛСЯ» + dead page (настраивается)

### Вася / Админ
- Написал «вася» (в любых формах) → **«АДМИН»**
- Написал «админ» → **«ВАСЯ»**

### Alan_Z (F6)
Каждые N сообщений — проявление крайнего интереса к темам: тренировки, лонгковид, фьючерсы, нейросети, жим дьявола. Случайная фраза из пула ~30 вариантов.

## 🛠 Настройка через `.env`

Все параметры живут в `.env`. После изменения — перезапусти бота.

### Общие

| Переменная | По умолчанию | Что делает |
|-----------|-------------|-----------|
| `API_TOKEN` | *(обязательно)* | Токен бота от @BotFather. Без него бот не запустится |
| `DB_PATH` | `local_database.db` | Путь к файлу SQLite. Меняй только если знаешь зачем |
| `MEDIA_BASE` | `media` | Корневая папка с медиафайлами |

### Пользователи

| Переменная | По умолчанию | Что делает |
|-----------|-------------|-----------|
| `KOSTIK_USER_ID` | `350803143` | Telegram ID Костика |
| `SLAVIK_USER_ID` | `479167456` | Telegram ID Славы |
| `ALAN_USER_ID` | `138811255` | Telegram ID Alan_Z |

### Костик (F7)

| Переменная | По умолчанию | Что делает |
|-----------|-------------|-----------|
| `KOSTIK_REPLY_PROBABILITY` | `1.0` | Вероятность ответа: `0.0` — никогда, `0.5` — в 50% случаев, `1.0` — всегда. Поставь `0` чтобы он заткнулся |

### Alan_Z (F6)

| Переменная | По умолчанию | Что делает |
|-----------|-------------|-----------|
| `ALAN_REPLY_INTERVAL` | `10` | Каждые N сообщений бот отвечает случайной фразой. `0` — отключает ответы |

### GIF-счётчик Славы (F3)

| Переменная | По умолчанию | Что делает |
|-----------|-------------|-----------|
| `GIF_INTERVAL` | `5` | Каждые N сообщений Славы — гифка. `0` — отключает |
| `GIF_PATH` | `media/slavic_chlen.mp4` | Путь к mp4-файлу. Можно заменить на свой |

### Dead Page (F2 v2)

| Переменная | По умолчанию | Что делает |
|-----------|-------------|-----------|
| `DEAD_PAGE_SOURCE_CHANNEL_USERNAME` | `d_pages` | Имя канала, репосты из которого триггерят dead page |
| `DEAD_PAGE_SOURCE_CHANNEL_ID` | `0` | ID канала-триггера. `0` — проверять только по username |
| `DEAD_PAGE_RELAY_CHANNEL_ID` | `4228645624` | ID приватного канала, откуда бот пересылает посты. Бот должен быть там админом |
| `DEAD_PAGE_COOLDOWN_SECONDS` | `10` | Минимальный интервал между dead page в одном чате. **`0` — отключает кулдаун**, dead page будет на каждый репост |
| `DEAD_PAGE_MAX_FORWARD_RETRIES` | `5` | Сколько раз пробовать разные посты из канала, если предыдущий не нашёлся |
| `DEAD_PAGE_POST_ON_JOIN` | `True` | Отправлять ли dead page при входе Славы. `False` — не отправлять |
| `DEAD_PAGE_CAPTION_MAX_CHARS` | `1024` | Максимальная длина подписи при fallback-отправке. Не меняй без необходимости |
| `DEAD_PAGE_DIR` | `media/dead_page` | Папка с картинками и текстами для fallback-режима |

## 🖼 Как работают dead page и fallback

### Основной режим (канал доступен)
1. Кто-то репостит сообщение из **@d_pages** в чат
2. Бот заходит в свой приватный канал (ID из `DEAD_PAGE_RELAY_CHANNEL_ID`)
3. Выбирает **случайный пост** и делает `forward` в чат
4. Картинка + текст приезжают одним постом, без разрыва

### Fallback (канал недоступен)
1. Бот берёт случайную **.jpg** и случайный **.txt** из папки `DEAD_PAGE_DIR`
2. Отправляет фото с подписью (до 1024 символов)
3. Если текст длиннее 1024 — шлёт второе сообщение с хвостом

Чтобы добавить новые dead page в fallback: просто кинь `.jpg` и `.txt` в `media/dead_page/`.

## 🎨 Как изменить текст ответов

Все ответы лежат в соответствующих файлах хендлеров. Каждый пул — это список строк. Добавь новую строку — расширишь пул.

### Костик — `handlers/kostik.py`

```python
KOSTIK_REPLIES = [
    "пошёл нахуй кринжатура ебаная",   # старый
    "твой новый insult сюда",           # добавь так
    "ещё один вариант",                 # и так
]
```

### Alan_Z — `handlers/alan.py`

```python
ALAN_REPLIES = [
    "так интересно, продолжай",        # старый
    "твоя новая фраза сюда",           # добавь так
]
```

### Слава (КУЧА / война / catch-all) — `handlers/slavik.py`

Ответы захардкожены в хендлерах. Меняй прямо в строках:

```python
await message.reply("ДАЛБАЕБ")       # ответ на «куча»
await message.reply("трясло ебаное")  # ответ на военные слова
await message.reply("пошёл нахуй")    # ответ на всё остальное
```

### Вася / Админ — `handlers/vasya.py`

```python
await message.reply("АДМИН")  # ответ на «вася»
await message.reply("ВАСЯ")   # ответ на «админ»
```

## 🐛 Известные нюансы

- Если `DEAD_PAGE_COOLDOWN_SECONDS=0`, кулдаун отключён — dead page шлётся на **каждый** репост из @d_pages, может быть спамно
- Слава иногда обижается, но всегда возвращается
- Костик не понимает, за что
- При отсутствии интернета уровень токсичности падает до нуля
- Для работы dead page в основном режиме бот **должен быть админом** приватного канала из `DEAD_PAGE_RELAY_CHANNEL_ID`
