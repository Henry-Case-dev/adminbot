# AdminBot — ваш персональный цифровой абьюзер

*Элегантное решение для автоматизации токсичного общения в чатах.*

**Версия:** v2.7.0 | **Тестов:** 252 | **Эпиков:** 12 (61 задача)

## 🚀 Возможности

### Слава: возвращение (F1)
Когда Слава заходит в чат (покинул → вернулся) — бот пишет **«ДОЛБОЕВ ВЕРНУЛСЯ»** и сразу шлёт dead page.

### Слава: Dead Page (F2 v2)
- Кто-то делает репост из канала **@d_pages** → бот выбирает случайный пост из своего приватного канала и пересылает его в чат
- Если канал недоступен → fallback: случайная картинка + текст из папки `media/dead_page/`
- При входе Славы в чат → «ДОЛБОЕВ ВЕРНУЛСЯ» + dead page (настраивается)

### Слава: чат-реакции (F3/F4)
Двухуровневая система:
- Просто написал → **«пошёл нахуй»** (catch-all)
- Упомянул «куча»/«кучи»/«кучу» → **«ДАЛБАЕБ»** (F4)
- Каждое 5-е сообщение → **GIF** со славным членом (F3)

### Слава: военный оповещатель (F5v2 — Epic 10)
Когда Слава пишет сообщение с военными ключевыми словами (дрон, БПЛА, ракета, опасность, тревога, убежище и т.д.) — бот отвечает случайной фразой из пула: «потрясись», «повизжи», «прячься под шконку быстрее», «закрой ушки и считай до десяти», «поплачь».

**Новое в v2 (Epic 10):**
- **Caption support**: исправлен баг — теперь детектит ключевые слова и в подписях к медиа (раньше только текст)
- **Channel reposts**: если **любой пользователь** репостит сообщение из каналов «ЧП Пермь» или «Радар БПЛА» — бот тоже отвечает
- **Расширенный словарь**: 90+ словоформ (склонения, синонимы)
- **Настраиваемый пул фраз**: меняется через `WAR_REPLIES` в `.env`
- **Детальное логгирование**: каждый этап (матч ключевого слова, матч канала, отправка, ошибка) пишется в Better Stack

### Alan_Z: автоответчик (F6)
Каждые N сообщений — проявление крайнего интереса к темам: тренировки, лонгковид, фьючерсы, нейросети, жим дьявола. Случайная фраза из пула ~40 вариантов.

### Alan_Z: приветственное видео (F7)
Когда Alan заходит в чат — бот отправляет случайное видео из папки `media/leha_greeting/` с подписью `@Alan_Z`. Срабатывает как на `ChatMemberUpdated` (основной детект), так и на `new_chat_members` (fallback). Кулдаун 10 секунд между повторами.

### Костик (F8)
На каждое сообщение — случайное оскорбление из пула. Вероятность настраивается: от «всегда» до «никогда».

### Вася / Админ
- Написал «вася» (в любых формах) → **«АДМИН»**
- Написал «админ» → **«ВАСЯ»**

### Тестовые команды для админа (Epic 10)
Две команды для ручного тестирования функциональности бота:

| Команда | Что делает | Где работает |
|---------|-----------|-------------|
| `/deadpage` | Триггерит отправку dead page (как будто кто-то репостнул @d_pages) | DM — всем, группа — только админу |
| `/alangreet` | Триггерит приветственное видео (как будто Alan зашёл в чат) | DM — всем, группа — только админу |

Сообщение с командой **автоматически удаляется** после выполнения. ID админа задаётся в `.env`: `ADMIN_USER_ID=5885953495`.

## 🖼 Как работают dead page и fallback

### Основной режим (канал доступен)
1. Кто-то репостит сообщение из **@d_pages** в чат
2. Бот заходит в свой приватный канал (ID из `DEAD_PAGE_RELAY_CHANNEL_ID`)
3. Выбирает **случайный пост** и делает `forward` в чат
4. Картинка + текст приезжают одним постом, без разрыва

### Fallback (канал недоступен)
1. Бот берёт случайную **.jpg** и случайный **.txt** из папки `DEAD_PAGE_DIR`
2. Отправляет фото с подписью (до 1024 символов)
3. Если текст длиннее 1024 — шлёт второе сообщение с хвостом

Чтобы добавить новые dead page в fallback: просто кинь `.jpg` и `.txt` в `media/dead_page/`.

## 🎨 Как изменить текст ответов

Все ответы лежат в соответствующих файлах хендлеров. Каждый пул — это список строк. Добавь новую строку — расширишь пул.

### Костик — `handlers/kostik.py`

```python
KOSTIK_REPLIES = [
    "пошёл нахуй кринжатура ебаная",   # старый
    "твой новый insult сюда",           # добавь так
    "ещё один вариант",                 # и так
]
```

### Alan_Z: автоответчик — `handlers/alan.py`

```python
ALAN_REPLIES = [
    "так интересно, продолжай",        # старый
    "твоя новая фраза сюда",           # добавь так
]
```

### Alan_Z: приветственное видео — `media/leha_greeting/`

Просто кинь `.mp4` файлы в папку `media/leha_greeting/`. Бот выберет случайный. Поддерживаются также `.avi`, `.mov`, `.webm`.

### Слава (КУЧА / война / catch-all) — `handlers/slavik.py`

Ответы захардкожены в хендлерах. Меняй прямо в строках:

```python
await message.reply("ДАЛБАЕБ")       # ответ на «куча»
await message.reply("трясло ебаное")  # ответ на военные слова
await message.reply("пошёл нахуй")    # ответ на всё остальное
```

### Вася / Админ — `handlers/vasya.py`

```python
await message.reply("АДМИН")  # ответ на «вася»
await message.reply("ВАСЯ")   # ответ на «админ»
```

## 📊 Мониторинг и логи (Better Stack)

Бот интегрирован с [Better Stack](https://betterstack.com/) — единой платформой для логов и ошибок. Два компонента:

| Компонент | Что делает | Библиотека |
|-----------|-----------|-----------|
| **Sentry** | Отлов необработанных исключений (ошибки Telegram API, aiogram, SQLite и любые другие) | `sentry-sdk` |
| **Logtail** | Стриминг всех логов уровня INFO+ в облако в реальном времени | `logtail-python` |

### Как посмотреть логи

1. Зайди на [betterstack.com](https://betterstack.com/) → войди в аккаунт команды
2. В левом меню выбери **Logs & Metrics** → **Live tail**
3. Все `logger.info(...)` / `logger.warning(...)` / `logger.error(...)` из кода прилетают сюда с тегом `source` и именем модуля (`bot`, `handlers.slavik`, `services.database` и т.д.)
4. Можно фильтровать по уровню (`level:error`), модулю (`name:bot`), или тексту сообщения

### Как посмотреть ошибки (Sentry)

1. На [betterstack.com](https://betterstack.com/) перейди в **Errors** (раздел Sentry-совместимых Issues)
2. Каждый необработанный `Exception` создаёт **Issue** со стектрейсом, контекстом (какой чат, какое сообщение), и тегами
3. Можно настроить алерты в Slack / Telegram / email — чтобы узнать о падении бота раньше, чем Слава напишет «чё бот молчит»

### Локально (без Better Stack)

Если переменные `SENTRY_DSN` и `LOGTAIL_SOURCE_TOKEN` не заданы в `.env` — бот работает как обычно. Логи пишутся только в консоль, ошибки не трекаются. Никаких лишних зависимостей или крашей.

### Тестовый smoke-тест

```bash
py tests/test_monitoring_smoke.py
```

Отправляет тестовый лог и тестовую ошибку в Better Stack. Полезно после настройки — убедиться, что креды рабочие.

## 🐛 Известные нюансы

- Если `DEAD_PAGE_COOLDOWN_SECONDS=0`, кулдаун отключён — dead page шлётся на **каждый** репост из @d_pages, может быть спамно
- Слава иногда обижается, но всегда возвращается
- Костик не понимает, за что
- При отсутствии интернета уровень токсичности падает до нуля
- Для работы dead page в основном режиме бот **должен быть админом** приватного канала из `DEAD_PAGE_RELAY_CHANNEL_ID`
- Приветственное видео Alan (F7) срабатывает только при **переходе статуса** (не был → стал участником). Если Alan уже в чате при запуске бота — видео не отправится. Используй `/alangreet` для ручного теста.
- Для поиска постов в relay-канале бот использует прогрессивный поиск по ID. Если канал вырос далеко за пределы последнего известного ID — поиск автоматически расширяется до 2000.
